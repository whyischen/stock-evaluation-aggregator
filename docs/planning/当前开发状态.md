## 最新进展（2026-06-10 下午）

### Docker 依赖隔离 ✅

**问题**：之前两个插件服务共用同一个 `Dockerfile.plugin`，导致：
- `tradingagents-svc` 和 `tradingagents-cn-svc` 都安装了 `tradingagents==0.7.0`（原版）
- TradingAgents-CN 应该使用 GitHub 版本（1.0.0rc0），但实际运行的是原版
- 违反了 Docker 依赖隔离的核心原则

**解决方案**：
- 创建 `Dockerfile.tradingagents` - 安装原版 `tradingagents==0.7.0`（PyPI）
- 创建 `Dockerfile.tradingagents-cn` - 从 GitHub 安装 `tradingagents==1.0.0rc0`
- 更新 `docker-compose.yml`，每个服务使用独立的 Dockerfile
- 为 TradingAgents-CN 添加 MongoDB 和 Redis 服务

**验证结果**：
```bash

Name: tradingagents
Version: 0.7.0

# tradingagents-cn-svc（CN 版）
Name: tradingagents
Version: 1.0.0rc0
```

### 修复 TradingAgents-CN propagate bug ✅

**问题**：TradingAgents-CN v1.0.0-preview 在执行评估时报错：
```
ValueError: dictionary update sequence element #0 has length 6; 2 is required
```

**根本原因**：
- LangGraph 的 `stream()` 方法返回的 `chunk` 格式与 TradingAgents-CN 代码期望的不一致
- 期望：`node_update` 是字典 `{key: value}`
- 实际：`node_update` 是序列（长度为 6 的元组/列表）
- 这是 TradingAgents-CN v1.0.0-preview 与当前 LangGraph 版本的兼容性问题

**解决方案**（由 Codex 完成）：
- 在 `plugins/tradingagents_common.py` 中添加 `_propagate()` 方法
- 通过 monkey-patch `graph.graph.stream`，将 LangGraph 返回的整份 state chunk 包装为 TradingAgents-CN 旧 `propagate()` 期望的 node update chunk
- 添加 `_normalize_cn_stream_chunk()` 函数处理 chunk 格式转换
- 原版 TradingAgents 不受影响
- 添加回归测试 `test_tradingagents_cn_normalizes_langgraph_state_stream_chunks`

## 历史完成项

### Web First MVP 骨架

- 建立 FastAPI 后端服务。
- 建立 SEA Core 轻量编排层。
- 建立静态 Web 评估页面。
- 建立标准化数据模型：
  - `EvaluationTask`
  - `PluginResult`
  - `EvalReport`
  - `PluginManifest`
  - `PluginHealth`
- 建立插件注册表。
- 建立插件编排器。
- 支持插件并发调用、超时保护和失败隔离。

### Mock 插件闭环

- 注册 3 个 Mock 插件：
  - `mock_bullish`
  - `mock_cautious`
  - `mock_failure`
- Mock 插件用于验证：
  - 多插件返回
  - 插件失败不阻塞整体评估
  - Web 页面展示成功和失败结果

### TradingAgents-CN 接入

- 新增 `tradingagents_cn` 插件适配器。
- A 股 ticker，如 `.SS` / `.SZ`，默认路由到 `TradingAgents-CN`。
- 真实调用默认关闭，避免意外产生 LLM API 费用。
- 开启真实调用需要：

```bash
export SEA_ENABLE_LIVE_TRADINGAGENTS=true
export OPENAI_API_KEY=...
```

### TradingAgents 接入

- 新增 `tradingagents` 插件适配器。
- 美股和港股默认路由到 `TradingAgents`。
- 与 TradingAgents-CN 共用薄适配逻辑：
  - 导入上游 `TradingAgentsGraph`
  - 调用 `propagate(ticker, date)`
  - 解析方向、置信度和摘要
  - 转换为 SEA 标准 `PluginResult`

### Web 插件选择

- Web 页面支持选择本次评估参与插件。
- 插件状态从 `/api/system/status` 读取。
- 评估请求会把选中的 `plugin_ids` 发送到 `/api/evaluate`。
