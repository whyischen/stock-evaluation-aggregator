# 当前开发状态

日期：2026-06-10

## 已完成

### Web First MVP 骨架

- 建立 FastAPI 后端服务。
- 建立 SEA Core 轻量编排层。
- 建立静态 Web 评估页面。
- 建立标准化数据模型：
  - `EvaluationTask`
  - `PluginResult`
  - `EvalReport`
  - `PluginManifest`
  - `PluginHealth`
- 建立插件注册表。
- 建立插件编排器。
- 支持插件并发调用、超时保护和失败隔离。

### Mock 插件闭环

- 注册 3 个 Mock 插件：
  - `mock_bullish`
  - `mock_cautious`
  - `mock_failure`
- Mock 插件用于验证：
  - 多插件返回
  - 插件失败不阻塞整体评估
  - Web 页面展示成功和失败结果

### TradingAgents-CN 接入

- 新增 `tradingagents_cn` 插件适配器。
- A 股 ticker，如 `.SS` / `.SZ`，默认路由到 `TradingAgents-CN`。
- 真实调用默认关闭，避免意外产生 LLM API 费用。
- 开启真实调用需要：

```bash
export SEA_ENABLE_LIVE_TRADINGAGENTS=true
export OPENAI_API_KEY=...
```

### TradingAgents 接入

- 新增 `tradingagents` 插件适配器。
- 美股和港股默认路由到 `TradingAgents`。
- 与 TradingAgents-CN 共用薄适配逻辑：
  - 导入上游 `TradingAgentsGraph`
  - 调用 `propagate(ticker, date)`
  - 解析方向、置信度和摘要
  - 转换为 SEA 标准 `PluginResult`

### Web 插件选择

- Web 页面支持选择本次评估参与插件。
- 插件状态从 `/api/system/status` 读取。
- 评估请求会把选中的 `plugin_ids` 发送到 `/api/evaluate`。

## 当前接口
