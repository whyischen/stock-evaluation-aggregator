### 1.2 核心设计原则

1. **无评估逻辑原则**：SEA Core 不包含任何股票评估算法，所有评估结论来自插件
2. **插件平等原则**：所有插件接口一致，添加/移除插件不影响其他插件和核心逻辑
3. **失败隔离原则**：单插件故障不影响整体任务，采用熔断 + 降级策略
4. **容器化优先原则**：生产环境以 Docker 容器运行为主，本地开发保留进程内模式

```python
# sea_core/dispatcher.py（双模式切换示意）
import httpx
from sea_core.config import Settings, PluginMode

class Dispatcher:
    def __init__(self, settings: Settings):
        self.settings = settings
        self._http_client = httpx.AsyncClient(timeout=300.0)  # 5 分钟超时

    async def dispatch(self, task: EvaluationTask) -> list[PluginResult]:
        if self.settings.plugin_mode == PluginMode.IN_PROCESS:
            return await self._dispatch_in_process(task)
        else:
            return await self._dispatch_http(task)

    async def _dispatch_http(self, task: EvaluationTask) -> list[PluginResult]:
        """并发 HTTP 调用各插件服务"""
        async def call_plugin(plugin_url: str, plugin_id: str):
            try:
                resp = await self._http_client.post(
                    f"{plugin_url}/evaluate",
                    json={"ticker": task.ticker, "eval_date": str(task.eval_date)},
                )
                resp.raise_for_status()
                return PluginResult(**resp.json())
            except Exception as e:
                return PluginResult(
                    plugin_id=plugin_id,
                    ticker=task.ticker,
                    eval_date=task.eval_date,
                    direction=None,
                    confidence=None,
                    summary="",
                    latency_ms=0,
                    error=str(e),
                )

        urls = self._get_enabled_plugin_urls(task)
        results = await asyncio.gather(
            *[call_plugin(url, pid) for pid, url in urls],
            return_exceptions=True,
        )
        return [r for r in results if isinstance(r, PluginResult)]
```

## 2. 插件适配器详细设计

### 2.1 TradingAgents 插件适配器

**适配复杂度：低**。TradingAgents 的 `propagate(ticker, date)` 接口与 SEA 的 `evaluate(ticker, date)` 几乎一一对应，适配层仅需做输出字段解析。

```python
# plugins/tradingagents/adapter.py
import time
from tradingagents.graph.trading_graph import TradingAgentsGraph
from sea_core.plugin_base import BaseEvaluatorPlugin, PluginResult, Direction

class TradingAgentsPlugin(BaseEvaluatorPlugin):
    plugin_id = "tradingagents"
    display_name = "TradingAgents (TauricResearch)"
    supported_markets = ["ALL"]

    def __init__(self, config: dict):
        self.config = config

    def is_available(self) -> bool:
        try:
            import tradingagents
            return True
        except ImportError:
            return False

    async def evaluate(self, ticker: str, eval_date: date, **kwargs) -> PluginResult:
        config = self._build_config(**kwargs)
        ta = TradingAgentsGraph(config=config)

        start = time.monotonic()
        try:
            state, _ = ta.propagate(ticker, str(eval_date))
            decision = self._parse_decision(state)
            latency = int((time.monotonic() - start) * 1000)

            return PluginResult(
                plugin_id=self.plugin_id,
                ticker=ticker,
                eval_date=eval_date,
                direction=decision["direction"],
                confidence=decision["confidence"],
                summary=decision["summary"],
                detail={"raw_state": state},
                latency_ms=latency,
            )
        except Exception as e:
            latency = int((time.monotonic() - start) * 1000)
            return PluginResult(
                plugin_id=self.plugin_id,
                ticker=ticker,
                eval_date=eval_date,
                direction=None,
                confidence=None,
                summary="",
                latency_ms=latency,
                error=str(e),
            )

    def _parse_decision(self, state: dict) -> dict:
        """
        从 TradingAgents 的 state 中解析标准化输出。

        state["final_trade_decision"] 结构示例:
        {
            "action": "BUY",          # BUY / HOLD / SELL
            "confidence": 0.78,       # 0.0 - 1.0
            "reasoning": "..."        # 分析理由
        }
        """
        decision = state.get("final_trade_decision", {})
        action = decision.get("action", "HOLD").upper()

        direction_map = {"BUY": Direction.BUY, "HOLD": Direction.HOLD, "SELL": Direction.SELL}
        direction = direction_map.get(action, Direction.HOLD)

        confidence = float(decision.get("confidence", 0.5))
        confidence = max(0.0, min(1.0, confidence))  # clamp

        # 取 risk_report 的 summary 作为简要结论
        summary = decision.get("reasoning", "")[:200]

        return {"direction": direction, "confidence": confidence, "summary": summary}

    def _build_config(self, **kwargs) -> dict:
        """构建 TradingAgents 原生配置"""
        return {
            "llm_provider": kwargs.get("llm_provider", self.config.get("llm_provider", "deepseek")),
            "deep_think": kwargs.get("research_depth", self.config.get("research_depth", "medium")),
            "api_key": self.config.get("api_key"),  # 从环境变量来
        }
```

### 2.3 Qlib 量化插件适配器（在线查询层）

**适配复杂度：中**。核心工作在于：
1. 前提条件管理（数据、模型、信号就绪检查）
2. Alpha 信号 → Direction 的阈值映射
3. 状态机管理

```python
# plugins/qlib_rdagent/qlib_adapter.py
import time
import pickle
from pathlib import Path
from datetime import date, datetime, timedelta
from enum import Enum

import pandas as pd
import qlib
from qlib.data import D
from qlib.data.dataset import DatasetH
from qlib.contrib.data.handler import Alpha158

class SignalStatus(str, Enum):
    SIGNAL_READY = "signal_ready"      # 信号就绪，可查询
    STALE = "stale"                     # 信号过期，建议更新
    RESEARCHING = "researching"         # 正在研究中
    NOT_INITIALIZED = "not_initialized" # 数据未初始化

class QlibQuantPlugin(BaseEvaluatorPlugin):
    plugin_id = "qlib-quant"
    display_name = "Qlib Quantitative (Microsoft)"
    supported_markets = ["A_SHARE", "US"]

    def __init__(self, config: dict):
        self.config = config
        self.qlib_data_dir = Path(config.get("qlib_data_dir", "~/.qlib/qlib_data")).expanduser()
        self.signal_cache_dir = Path(config.get("signal_cache_dir", "~/.qlib/signals")).expanduser()

        # 配置项
        self.alpha_buy_threshold = config.get("alpha_buy_threshold", 0.02)    # +2%
        self.alpha_sell_threshold = config.get("alpha_sell_threshold", -0.02)  # -2%
        self.signal_stale_days = config.get("signal_stale_days", 7)            # 7 天过期

        # 状态
        self._signal_status = SignalStatus.NOT_INITIALIZED
        self._last_signal_update: Optional[datetime] = None

    def is_available(self) -> bool:
        """检查 Qlib 数据 + 信号是否就绪"""
        if not self.qlib_data_dir.exists():
            return False
        # 检查是否有预计算的信号文件
        signal_files = list(self.signal_cache_dir.glob("signals_*.pkl"))
        if not signal_files:
            return False
        self._signal_status = SignalStatus.SIGNAL_READY
        # 检查是否过期
        latest = max(signal_files, key=lambda f: f.stat().st_mtime)
        self._last_signal_update = datetime.fromtimestamp(latest.stat().st_mtime)
        days_since = (datetime.now() - self._last_signal_update).days
        if days_since > self.signal_stale_days:
            self._signal_status = SignalStatus.STALE
        return True

    async def evaluate(self, ticker: str, eval_date: date, **kwargs) -> PluginResult:
        start = time.monotonic()

        try:
            # 1. 确保 Qlib 已初始化
            self._ensure_qlib_initialized()

            # 2. 查询该 ticker 在目标日期的 Alpha 信号
            alpha_signal = self._query_signal(ticker, eval_date)

            # 3. Alpha → Direction 映射
            direction, confidence = self._alpha_to_direction(alpha_signal)

            # 4. 计算信号年龄
            signal_age = None
            if self._last_signal_update:
                signal_age = (datetime.now() - self._last_signal_update).days

            latency = int((time.monotonic() - start) * 1000)

            return PluginResult(
                plugin_id=self.plugin_id,
                ticker=ticker,
                eval_date=eval_date,
                direction=direction,
                confidence=confidence,
                summary=self._build_summary(alpha_signal, direction, signal_age),
                detail={
                    "alpha_signal": alpha_signal,
                    "signal_status": self._signal_status.value,
                    "last_signal_update": str(self._last_signal_update),
                },
                latency_ms=latency,
                signal_age_days=signal_age,
            )

        except Exception as e:
            latency = int((time.monotonic() - start) * 1000)
            return PluginResult(
                plugin_id=self.plugin_id,
                ticker=ticker,
                eval_date=eval_date,
                direction=None,
                confidence=None,
                summary="",
                latency_ms=latency,
                error=str(e),
                signal_age_days=(
                    (datetime.now() - self._last_signal_update).days
                    if self._last_signal_update else None
                ),
            )

    def _ensure_qlib_initialized(self):
        """确保 Qlib 已初始化（懒加载）"""
        if not hasattr(self, '_qlib_initialized'):
            qlib.init(
                provider_uri=str(self.qlib_data_dir),
                region="cn" if "CSI" in str(self.qlib_data_dir) else "us",
            )
            self._qlib_initialized = True

    def _query_signal(self, ticker: str, eval_date: date) -> float:
        """
        从预计算的信号文件中查询指定 ticker 的 Alpha 信号。

        信号文件结构（pickle）:
        {
            "date": "2026-06-01",
            "signals": {"600519.SS": 0.035, "000858.SZ": -0.012, ...},
            "model_metrics": {"ic": 0.045, "icir": 0.52, "sharpe": 1.2}
        }
        """
        signal_files = sorted(self.signal_cache_dir.glob("signals_*.pkl"))
        if not signal_files:
            raise FileNotFoundError("No signal cache found. Run research task first.")

        # 找 eval_date 之前的最新信号文件
        target_date = pd.Timestamp(eval_date)
        best_file = None
        for f in signal_files:
            file_date = pd.Timestamp(f.stem.replace("signals_", ""))
            if file_date <= target_date:
                best_file = f

        if best_file is None:
            raise ValueError(f"No signal available for date {eval_date}")

        with open(best_file, "rb") as fh:
            signal_data = pickle.load(fh)

        return signal_data["signals"].get(ticker, 0.0)

    def _alpha_to_direction(self, alpha: float) -> tuple[Direction, float]:
        """
        Alpha 信号 → Direction 映射。

        Alpha > alpha_buy_threshold  → BUY
        Alpha < alpha_sell_threshold → SELL
        否则 → HOLD

        置信度基于 Alpha 的绝对值映射到 [0.5, 1.0] 区间：
        - Alpha = 阈值时置信度 0.5（最不确定）
        - Alpha 绝对值越大置信度越高
        """
        if alpha > self.alpha_buy_threshold:
            direction = Direction.BUY
            # 越超过阈值，置信度越高
            confidence = min(1.0, 0.5 + (alpha - self.alpha_buy_threshold) * 10)
        elif alpha < self.alpha_sell_threshold:
            direction = Direction.SELL
            confidence = min(1.0, 0.5 + (abs(alpha) - abs(self.alpha_sell_threshold)) * 10)
        else:
            direction = Direction.HOLD
            # 在阈值区间内，越接近 0 越确定是 HOLD
            distance = min(abs(alpha - self.alpha_buy_threshold), abs(alpha - self.alpha_sell_threshold))
            confidence = 0.5 + distance * 5

        return direction, round(confidence, 2)

    def _build_summary(self, alpha: float, direction: Direction, signal_age: int) -> str:
        direction_cn = {Direction.BUY: "看多", Direction.HOLD: "中性", Direction.SELL: "看空"}
        age_str = f"（信号 {signal_age} 天前）" if signal_age else ""
        return f"Alpha={alpha:.4f}, {direction_cn.get(direction, '未知')}{age_str}"
```

### 2.4 RD-Agent 研究任务封装（离线研究层）

```python
# plugins/qlib_rdagent/rdagent_research.py
import subprocess
import json
import asyncio
from datetime import datetime
from pathlib import Path
from enum import Enum

class ResearchStatus(str, Enum):
    PENDING = "pending"
    RUNNING = "running"
    COMPLETED = "completed"
    FAILED = "failed"

class ResearchTask:
    """封装一次 RD-Agent 研究任务"""

    def __init__(self, task_id: str, config: dict):
        self.task_id = task_id
        self.config = config
        self.status = ResearchStatus.PENDING
        self.progress = ""
        self.started_at: Optional[datetime] = None
        self.completed_at: Optional[datetime] = None
        self.result_path: Optional[Path] = None
        self._process: Optional[subprocess.Popen] = None

    async def run(self):
        """
        以子进程方式运行 rdagent fin_quant 命令。

        RD-Agent 内部会自动：
        1. 调用 Qlib 数据接口加载数据
        2. Research 阶段：提出因子假设
        3. Development 阶段：Co-STEER 生成因子代码
        4. Qlib 回测验证
        5. 迭代优化（约 10 轮）
        6. 输出因子集 + 训练好的模型
        """
        self.status = ResearchStatus.RUNNING
        self.started_at = datetime.now()

        cmd = [
            "rdagent", "fin_quant",
            "--market", self.config.get("market", "CSI300"),
            "--train-start", self.config.get("train_start", "2008-01-01"),
            "--train-end", self.config.get("train_end", "2014-12-31"),
            "--valid-start", self.config.get("valid_start", "2015-01-01"),
            "--valid-end", self.config.get("valid_end", "2016-12-31"),
            "--test-start", self.config.get("test_start", "2017-01-01"),
            "--test-end", self.config.get("test_end", "2020-12-31"),
            "--output-dir", str(self.config.get("output_dir", "/workspace/research_output")),
        ]

        try:
            self._process = await asyncio.create_subprocess_exec(
                *cmd,
                stdout=asyncio.subprocess.PIPE,
                stderr=asyncio.subprocess.PIPE,
                env={
                    **__import__("os").environ,
                    "OPENAI_API_KEY": self.config.get("openai_api_key", ""),
                    "DEEPSEEK_API_KEY": self.config.get("deepseek_api_key", ""),
                },
            )

            # 逐行读取输出，更新进度
            async for line in self._process.stdout:
                line_str = line.decode().strip()
                self.progress = line_str
                # RD-Agent 输出中通常有明确的进度标记
                if "Iteration" in line_str:
                    self.progress = f"因子回测验证中: {line_str}"

            await self._process.wait()

            if self._process.returncode == 0:
                self.status = ResearchStatus.COMPLETED
                self.completed_at = datetime.now()
                await self._post_process_results()
            else:
                self.status = ResearchStatus.FAILED
                stderr = await self._process.stderr.read()
                self.progress = f"Failed: {stderr.decode()[:500]}"

        except Exception as e:
            self.status = ResearchStatus.FAILED
            self.progress = f"Exception: {str(e)}"

    async def _post_process_results(self):
        """
        研究完成后：
        1. 用新发现的因子 + 模型对全量股票池跑预测
        2. 生成新的信号文件（signals_{date}.pkl）
        3. 更新信号库（替换或追加）
        """
        output_dir = Path(self.config.get("output_dir", "/workspace/research_output"))
        signal_dir = Path(self.config.get("signal_cache_dir", "/root/.qlib/signals"))
        signal_dir.mkdir(parents=True, exist_ok=True)

        # 调用 Qlib 批量预测脚本
        predict_script = Path(__file__).parent / "scripts" / "batch_predict.py"
        result = await asyncio.create_subprocess_exec(
            "python", str(predict_script),
            "--model-dir", str(output_dir / "models"),
            "--factor-dir", str(output_dir / "factors"),
            "--qlib-data-dir", str(self.config.get("qlib_data_dir", "/root/.qlib/qlib_data")),
            "--output", str(signal_dir / f"signals_{datetime.now().strftime('%Y%m%d')}.pkl"),
        )
        await result.wait()

        self.result_path = signal_dir
```

## 3. Qlib + RD-Agent 双层架构设计

### 3.1 双层架构设计理由

Qlib 和 RD-Agent 共享 Python 环境和数据目录，放在同一容器中最合理：
- RD-Agent 安装时自动安装 Qlib 作为依赖
- 两者使用同一个 `~/.qlib/qlib_data` 数据目录
- 分开放反而带来数据路径同步的麻烦
- 统一对外暴露接口，对内管理复杂协作关系

### 3.3 插件状态机

```
                    ┌──────────────┐
                    │ NOT_INIT     │  数据目录为空
                    │ _IALIZED     │
                    └──────┬───────┘
                           │ 执行 init_qlib_data.py
                           │ + train_model.py
                           │ + batch_predict.py
                           ▼
                    ┌──────────────┐
          ┌────────│ SIGNAL_READY │  信号就绪，正常响应 evaluate()
          │        └──────┬───────┘
          │               │ 超过 signal_stale_days（默认 7 天）
          │               ▼
          │        ┌──────────────┐
          │        │    STALE     │  信号过期，evaluate() 仍可用
          │        └──────┬───────┘  但返回降级标记
          │               │ 触发 /research/trigger
          │               ▼
          │        ┌──────────────┐
          │        │ RESEARCHING  │  RD-Agent 运行中
          │        └──────┬───────┘  evaluate() 降级到上期信号
          │               │ 研究完成 → batch_predict.py
          │               ▼
          │        ┌──────────────┐
          └───────▶│ SIGNAL_READY │  循环
                   └──────────────┘

状态转换触发条件：
- NOT_INITIALIZED → SIGNAL_READY：手动执行初始化脚本
- SIGNAL_READY → STALE：距上次信号更新超过 signal_stale_days
- STALE → RESEARCHING：手动或定时触发研究任务
- STALE → SIGNAL_READY：研究任务完成，batch_predict 生成新信号
- RESEARCHING → SIGNAL_READY：研究成功完成
- RESEARCHING → STALE：研究失败，回退（仍有上期信号可用）
```

### 3.4 evaluate() 在不同状态下的行为

| 状态 | evaluate() 行为 | 返回内容 |
|---|---|---|
| SIGNAL_READY | 正常查询最新信号 | direction + confidence + signal_age |
| STALE | 查询上期信号（仍可用） | direction + confidence + signal_age + error 字段标注"信号已过期 N 天" |
| RESEARCHING | 查询上期信号 | direction + confidence + signal_age + error 字段标注"研究中，使用上期信号" |
| NOT_INITIALIZED | 返回空 | error="数据未初始化，请先执行初始化脚本" |

## 7. DooD 模式详解

### 7.1 为什么需要 DooD

RD-Agent 在运行时会动态创建 Docker 容器（`local_qlib:latest`）来隔离执行生成的因子代码。当 RD-Agent 本身运行在 Docker 容器内时，需要让容器内的 Docker 命令操作宿主机的 Docker Daemon。

### 7.2 DinD vs DooD

| 维度 | DinD（Docker-in-Docker） | DooD（Docker-outside-of-Docker） |
|---|---|---|
| 原理 | 容器内启动独立 Docker Daemon | 挂载宿主机 Docker socket |
| 权限 | 需要 `--privileged` | 仅需 socket 文件访问 |
| 安全性 | 低（特权模式，容器逃逸风险） | 较高（仅操作宿主机 Docker API） |
| 资源 | 两份 Docker Daemon 开销 | 无额外开销 |
| 创建的容器 | 嵌套容器（子容器） | 兄弟容器（与当前容器同级） |
| 推荐度 | ❌ 不推荐 | ✅ 推荐 |

### 7.3 DooD 实现

```yaml
# docker-compose.yml 中 qlib-rdagent-svc 的关键配置
services:
  qlib-rdagent-svc:
    volumes:
      # DooD: 挂载宿主机 Docker socket
      - /var/run/docker.sock:/var/run/docker.sock
      # RD-Agent 工作目录（必须宿主机可识别路径）
      - rdagent_workspace:/workspace
    environment:
      # 告知 RD-Agent 使用宿主机视角的路径
      - DOCKER_HOST=unix:///var/run/docker.sock
      - RDAGENT_WORKSPACE_HOST_PATH=/var/lib/docker/volumes/sea_rdagent_workspace/_data
```

### 7.4 卷路径一致性问题及解决方案

**问题**：DooD 模式下，当 RD-Agent 通过 Docker API 创建 `local_qlib` 容器并挂载工作目录时，bind mount 的源路径必须从**宿主机 Docker Daemon** 的视角可解析，而不是从容器内部视角。

**示例**：
- 容器内部路径：`/workspace/research_output`
- 宿主机实际路径：`/var/lib/docker/volumes/sea_rdagent_workspace/_data/research_output`
- 如果 RD-Agent 告诉 Docker "把 `/workspace/research_output` 挂载进去"，宿主机找不到这个路径，挂载失败

**解决方案**：

方案 1（推荐）：使用 Named Volume 并在容器内使用与宿主机一致的路径：
```yaml
volumes:
  rdagent_workspace:
    driver: local
    driver_opts:
      type: none
      o: bind
      device: /home/user/sea/rdagent_workspace  # 宿主机绝对路径

services:
  qlib-rdagent-svc:
    volumes:
      - /home/user/sea/rdagent_workspace:/home/user/sea/rdagent_workspace
    environment:
      - RDAGENT_WORKSPACE=/home/user/sea/rdagent_workspace
```

方案 2：通过环境变量传递宿主机路径：
```python
# 在 ResearchTask.run() 中，传递宿主机视角的路径
host_workspace = os.environ.get("RDAGENT_WORKSPACE_HOST_PATH", "/workspace")

    "rdagent", "fin_quant",
    "--output-dir", host_workspace + "/research_output",
    ...

```

### 7.5 Mac 上的特殊处理

Docker Desktop on Mac 在后台运行一个 Linux VM。Docker socket 在 Mac 上有两个版本：

- `/var/run/docker.sock`：符号链接，可能有兼容性问题
- `/var/run/docker.sock.raw`：直接 socket，推荐在 Mac 上使用

```yaml
# Mac 上的配置
services:
  qlib-rdagent-svc:
    volumes:
      # Mac 上使用 .raw socket
      - /var/run/docker.sock.raw:/var/run/docker.sock
```

## 10. 硬件需求与平台兼容性

### 10.1 各组件资源需求

| 组件 | CPU | 内存（闲时） | 内存（峰值） | 磁盘 | 平台 |
|---|---|---|---|---|---|
| sea-core | 1 核 | 256MB | 512MB | 1GB | Linux / macOS |
| tradingagents-svc | 1 核 | 512MB | 2GB | 2GB | Linux / macOS |
| tradingagents-cn-svc | 1 核 | 512MB | 2GB | 2GB | Linux / macOS |
| qlib-rdagent-svc（仅查询） | 1 核 | 1GB | 2GB | 10GB | Linux / macOS |
| qlib-rdagent-svc（含训练） | 2-4 核 | 2GB | 8-12GB | 20GB | Linux（推荐）/macOS（可尝试） |
| qlib-rdagent-svc（含 RD-Agent） | 2-4 核 | 2GB | 8-12GB | 25GB | **仅 Linux** |
| finrobot-svc（候选） | 1 核 | 512MB | 2GB | 2GB | Linux / macOS |

### 10.2 Mac Mini (16GB / 256GB) 可行性总结

| 场景 | 可行性 | 说明 |
|---|---|---|
| SEA Core + LLM 插件（TradingAgents 系列） | ✅ 完全可行 | 仅需 LLM API 调用，无需本地 GPU |
| Qlib 数据下载 + 初始训练 | ✅ 可行 | LightGBM 训练 CSI300 峰值约 8-12GB |
| Qlib 日常信号查询 | ✅ 完全可行 | 闲时内存 < 2GB |
| Qlib 模型重训练（周度/月度） | ✅ 可行 | 安排在闲时运行 |
| RD-Agent 研究任务 | ⚠️ 需变通 | 非官方支持 macOS；建议方案二（云上运行） |
| 全量 A 股日线数据 | ✅ 可行 | ~15-20GB，磁盘够用 |
| 分钟线数据 | ❌ 不可行 | ~80-150GB，磁盘不够 |

### 10.3 Mac 上运行 Qlib 的注意事项

1. **Apple Silicon（M 系列）**：Qlib 大部分组件可在 ARM 架构的原生 Python 中运行，LightGBM 等 C 扩展有 ARM 版本
2. **数据下载**：CSI300 数据约 3-5GB，下载时间取决于网络
3. **模型训练**：LightGBM 在 M 系列芯片上性能良好；LSTM/Transformer 可能较慢但可运行
4. **不建议在 Mac 上跑 RD-Agent**：即使通过 Docker Desktop 可行，兼容性问题可能耗费大量调试时间

### 10.4 推荐的 Mac Mini 使用策略

```
Mac Mini（本地，日常运行）:
├── sea-core                  ✅ 随时运行
├── tradingagents-svc         ✅ 随时运行
├── tradingagents-cn-svc      ✅ 随时运行
├── qlib-rdagent-svc          ✅ 查询层随时运行
│   └── 训练：周末后台自动运行
└── RD-Agent 研究任务          ☁️ 云服务器（按需启动，跑完关停）

云服务器（按需，仅 RD-Agent 运行时）:
├── 规格：2 核 8GB 内存 Linux ECS
├── 频率：每周/每月运行一次
├── 流程：触发研究 → rsync 结果到 Mac → 关闭 ECS
└── 成本：按量付费，每次数小时约 ¥5-15
```
